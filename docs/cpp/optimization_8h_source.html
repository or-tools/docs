152 // If we have an "at most one can be false" between literals with a positive
153 // cost, you then know that at least n - 1 will contribute to the cost, and
154 // you can increase the objective lower bound. This is the same as having
155 // a real "at most one" constraint on the negation of such literals.
156 //
157 // This detects such "at most ones" and rewrite the objective accordingly.
158 // For each at most one, the rewrite create a new Boolean variable and update
159 // the cost so that the trivial objective lower bound reflect the increase.
160 //
161 // TODO(user) : Code that as a general presolve rule? I am not sure adding
162 // the extra Booleans is always a good idea though. Especially since the LP
163 // will see the same lower bound that what is computed by this.
164 void PresolveObjectiveWithAtMostOne(std::vector<Literal>* literals,
165 std::vector<Coefficient>* coefficients,
