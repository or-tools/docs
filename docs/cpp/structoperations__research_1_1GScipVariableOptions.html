◆ keep_alive

bool operations_research::GScipVariableOptions::keep_alive = true

gSCIP options. If keep_alive=true, the returned variable will not to be freed until after ~GScip() is called. Otherwise, the returned variable could be freed internally by SCIP at any point, and it is not safe to hold a reference to the returned variable.

The primary reason to set keep_alive=false is if you are adding many variables in a callback (in branch and price), and you expect that most of them will be deleted.
