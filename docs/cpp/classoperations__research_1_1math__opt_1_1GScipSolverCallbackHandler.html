Detailed Description

Handler for user callbacks for GScipSolver.

It deals with solve interruption when the user request it or when an error occurs during the call of the user callback. Any such error is returned by Flush().

Returns a non null handler if needed (there are supported events that we register to).

The caller will also have to use MessageHandler() when calling GScip::Solve() when the result is not nullptr.

At the end of the solve, Flush() must be called (when everything else succeeded) to make the final user callback calls and return the first error that occurred when calling the user callback.
