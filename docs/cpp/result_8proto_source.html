27// Problem feasibility status as claimed by the solver (solver is not required
28// to return a certificate for the claim).

31 FEASIBILITY_STATUS_UNSPECIFIED = 0;
32
33 // Solver does not claim a status.
34 FEASIBILITY_STATUS_UNDETERMINED = 1;

36 // Solver claims the problem is feasible.
37 FEASIBILITY_STATUS_FEASIBLE = 2;
38
39 // Solver claims the problem is infeasible.
40 FEASIBILITY_STATUS_INFEASIBLE = 3;

43// Feasibility status of the primal problem and its dual (or the dual of a
44// continuous relaxation) as claimed by the solver. The solver is not required
45// to return a certificate for the claim (e.g. the solver may claim primal
46// feasibility without returning a primal feasible solutuion). This combined
47// status gives a comprehensive description of a solver's claims about
48// feasibility and unboundedness of the solved problem. For instance,
49//
50// * a feasible status for primal and dual problems indicates the primal is
51// feasible and bounded and likely has an optimal solution (guaranteed for
52// problems without non-linear constraints).
53// * a primal feasible and a dual infeasible status indicates the primal
54// problem is unbounded (i.e. has arbitrarily good solutions).
55//
56// Note that a dual infeasible status by itself (i.e. accompanied by an
57// undetermined primal status) does not imply the primal problem is unbounded as
58// we could have both problems be infeasible. Also, while a primal and dual
59// feasible status may imply the existence of an optimal solution, it does not
60// guarantee the solver has actually found such optimal solution.
61message ProblemStatusProto {
62 // Status for the primal problem.
63 FeasibilityStatusProto primal_status = 1;
64
65 // Status for the dual problem (or for the dual of a continuous relaxation).
66 FeasibilityStatusProto dual_status = 2;
67
68 // If true, the solver claims the primal or dual problem is infeasible, but
69 // it does not know which (or if both are infeasible). Can be true only when
70 // primal_problem_status = dual_problem_status = kUndetermined. This extra
71 // information is often needed when preprocessing determines there is no
72 // optimal solution to the problem (but can't determine if it is due to
73 // infeasibility, unboundedness, or both).
74 bool primal_or_dual_infeasible = 3;
75}

77message SolveStatsProto {
78 // Elapsed wall clock time as measured by math_opt, roughly the time inside
79 // Solver::Solve(). Note: this does not include work done building the model.
80 google.protobuf.Duration solve_time = 1;
81
82 // Solver claims the optimal value is equal or better (smaller for
83 // minimization and larger for maximization) than best_primal_bound up to the
84 // solvers primal feasibility tolerance (see warning below):
85 // * best_primal_bound is trivial (+inf for minimization and -inf
86 // maximization) when the solver does not claim to have such bound.
87 // * best_primal_bound can be closer to the optimal value than the objective
88 // of the best primal feasible solution. In particular, best_primal_bound
89 // may be non-trivial even when no primal feasible solutions are returned.
90 // Warning: The precise claim is that there exists a primal solution that:
91 // * is numerically feasible (i.e. feasible up to the solvers tolerance), and
92 // * has an objective value best_primal_bound.
93 // This numerically feasible solution could be slightly infeasible, in which
94 // case best_primal_bound could be strictly better than the optimal value.
95 // Translating a primal feasibility tolerance to a tolerance on
96 // best_primal_bound is non-trivial, specially when the feasibility tolerance
97 // is relatively large (e.g. when solving with PDLP).
98 double best_primal_bound = 2;
99
100 // Solver claims the optimal value is equal or worse (larger for
101 // minimization and smaller for maximization) than best_dual_bound up to the
102 // solvers dual feasibility tolerance (see warning below):
103 // * best_dual_bound is trivial (-inf for minimization and +inf
104 // maximization) when the solver does not claim to have such bound.
105 // Similarly to best_primal_bound, this may happen for some solvers even
106 // when returning optimal. MIP solvers will typically report a bound even
107 // if it is imprecise.
108 // * for continuous problems best_dual_bound can be closer to the optimal
109 // value than the objective of the best dual feasible solution. For MIP
110 // one of the first non-trivial values for best_dual_bound is often the
111 // optimal value of the LP relaxation of the MIP.
112 // * best_dual_bound should be better (smaller for minimization and larger
113 // for maximization) than best_primal_bound up to the solvers tolerances
114 // (see warning below).
115 // Warning:
116 // * For continuous problems, the precise claim is that there exists a
117 // dual solution that:
118 // * is numerically feasible (i.e. feasible up to the solvers tolerance),
119 // and
120 // * has an objective value best_dual_bound.
121 // This numerically feasible solution could be slightly infeasible, in
122 // which case best_dual_bound could be strictly worse than the optimal
123 // value and best_primal_bound. Similar to the primal case, translating a
124 // dual feasibility tolerance to a tolerance on best_dual_bound is
125 // non-trivial, specially when the feasibility tolerance is relatively
126 // large. However, some solvers provide a corrected version of
127 // best_dual_bound that can be numerically safer. This corrected version
128 // can be accessed through the solver's specific output (e.g. for PDLP,
129 // pdlp_output.convergence_information.corrected_dual_objective).
130 // * For MIP solvers, best_dual_bound may be associated to a dual solution
131 // for some continuous relaxation (e.g. LP relaxation), but it is often a
132 // complex consequence of the solvers execution and is typically more
133 // imprecise than the bounds reported by LP solvers.
134 double best_dual_bound = 3;

137 ProblemStatusProto problem_status = 4;
138
139 int64 simplex_iterations = 5;
140
141 int64 barrier_iterations = 6;
142

150// The reason a call to Solve() terminates.
151enum TerminationReasonProto {
152 TERMINATION_REASON_UNSPECIFIED = 0;
153
154 // A provably optimal solution (up to numerical tolerances) has been found.
155 TERMINATION_REASON_OPTIMAL = 1;
156
157 // The primal problem has no feasible solutions.
158 TERMINATION_REASON_INFEASIBLE = 2;
159
160 // The primal problem is feasible and arbitrarily good solutions can be
161 // found along a primal ray.
162 TERMINATION_REASON_UNBOUNDED = 3;
163
164 // The primal problem is either infeasible or unbounded. More details on the
165 // problem status may be available in solve_stats.problem_status. Note that
166 // Gurobi's unbounded status may be mapped here.
167 TERMINATION_REASON_INFEASIBLE_OR_UNBOUNDED = 4;
168
169 // The problem was solved to one of the criteria above (Optimal, Infeasible,
170 // Unbounded, or InfeasibleOrUnbounded), but one or more tolerances was not
171 // met. Some primal/dual solutions/rays be present, but either they will be
172 // slightly infeasible, or (if the problem was nearly optimal) their may be
173 // a gap between the best solution objective and best objective bound.
174 //
175 // Users can still query primal/dual solutions/rays and solution stats, but
176 // they are responsible for dealing with the numerical imprecision.
177 TERMINATION_REASON_IMPRECISE = 5;
178
179 // The optimizer reached some kind of limit and a primal feasible solution
180 // is returned. See SolveResultProto.limit_detail for detailed description of
181 // the kind of limit that was reached.
182 TERMINATION_REASON_FEASIBLE = 9;
183
184 // The optimizer reached some kind of limit and it did not find a primal
185 // feasible solution. See SolveResultProto.limit_detail for detailed
186 // description of the kind of limit that was reached.
187 TERMINATION_REASON_NO_SOLUTION_FOUND = 6;
188
189 // The algorithm stopped because it encountered unrecoverable numerical
190 // error. No solution information is available.
191 TERMINATION_REASON_NUMERICAL_ERROR = 7;
192
193 // The algorithm stopped because of an error not covered by one of the
194 // statuses defined above. No solution information is available.
195 TERMINATION_REASON_OTHER_ERROR = 8;
196}

198// When a Solve() stops early with TerminationReasonProto FEASIBLE or
199// NO_SOLUTION_FOUND, the specific limit that was hit.
200enum LimitProto {
201 // Used as a null value when we terminated not from a limit (e.g.
202 // TERMINATION_REASON_OPTIMAL).
203 LIMIT_UNSPECIFIED = 0;
204
205 // The underlying solver does not expose which limit was reached.
206 LIMIT_UNDETERMINED = 1;
207
208 // An iterative algorithm stopped after conducting the maximum number of
209 // iterations (e.g. simplex or barrier iterations).
210 LIMIT_ITERATION = 2;
211
212 // The algorithm stopped after a user-specified computation time.
213 LIMIT_TIME = 3;
214
215 // A branch-and-bound algorithm stopped because it explored a maximum number
216 // of nodes in the branch-and-bound tree.
217 LIMIT_NODE = 4;
218
219 // The algorithm stopped because it found the required number of solutions.
220 // This is often used in MIPs to get the solver to return the first feasible
221 // solution it encounters.
222 LIMIT_SOLUTION = 5;
223
224 // The algorithm stopped because it ran out of memory.
225 LIMIT_MEMORY = 6;
226
227 // The solver was run with a cutoff (e.g. SolveParameters.cutoff_limit was
228 // set) on the objective, indicating that the user did not want any solution
229 // worse than the cutoff, and the solver concluded there were no solutions at
230 // least as good as the cutoff. Typically no further solution information is
231 // provided.
232 LIMIT_CUTOFF = 12;
233
234 // The algorithm stopped because it either found a solution or a bound better
235 // than a limit set by the user (see SolveParameters.objective_limit and
236 // SolveParameters.best_bound_limit).
237 LIMIT_OBJECTIVE = 7;
238
239 // The algorithm stopped because the norm of an iterate became too large.
240 LIMIT_NORM = 8;
241
242 // The algorithm stopped because of an interrupt signal or a user interrupt
243 // request.
244 LIMIT_INTERRUPTED = 9;
245
246 // The algorithm stopped because it was unable to continue making progress
247 // towards the solution.
248 LIMIT_SLOW_PROGRESS = 10;
249
250 // The algorithm stopped due to a limit not covered by one of the above. Note
251 // that LIMIT_UNDETERMINED is used when the reason cannot be determined, and
252 // LIMIT_OTHER is used when the reason is known but does not fit into any of
253 // the above alternatives.
254 //
255 // TerminationProto.detail may contain additional information about the limit.
256 LIMIT_OTHER = 11;
257}

259// Bounds on the optimal objective value.
260message ObjectiveBoundsProto {
261 // Solver claims there exists a primal solution that is numerically feasible
262 // (i.e. feasible up to the solvers tolerance), and whose objective value is
263 // primal_bound.
264 //
265 // The optimal value is equal or better (smaller for min objectives and larger
266 // for max objectives) than primal_bound, but only up to solver-tolerances.
267 double primal_bound = 2;
268
269 // Solver claims there exists a dual solution that is numerically feasible
270 // (i.e. feasible up to the solvers tolerance), and whose objective value is
271 // dual_bound.
272 //
273 // For MIP solvers, the associated dual problem may be some continuous
274 // relaxation (e.g. LP relaxation), but it is often an implicitly defined
275 // problem that is a complex consequence of the solvers execution. For both
276 // continuous and MIP solvers, the optimal value is equal or worse (larger for
277 // min objective and smaller for max objectives) than dual_bound, but only up
278 // to solver-tolerances. Some continuous solvers provide a numerically safer
279 // dual bound through solver's specific output (e.g. for PDLP,
280 // pdlp_output.convergence_information.corrected_dual_objective).
281 double dual_bound = 3;
282}

313// The contract of when primal/dual solutions/rays is complex, see
314// termination_reasons.md for a complete description.
315//
316// Until an exact contract is finalized, it is safest to simply check if a
317// solution/ray is present rather than relying on the termination reason.
318message SolveResultProto {
319 // The reason the solver stopped.
320 TerminationProto termination = 2;
321
322 // Basic solutions use, as of Nov 2021:
323 // * All convex optimization solvers (LP, convex QP) return only one
324 // solution as a primal dual pair.
325 // * Only MI(Q)P solvers return more than one solution. MIP solvers do not
326 // return any dual information, or primal infeasible solutions. Solutions
327 // are returned in order of best primal objective first. Gurobi solves
328 // nonconvex QP (integer or continuous) as MIQP.
329
330 // The general contract for the order of solutions that future solvers should
331 // implement is to order by:
332 // 1. The solutions with a primal feasible solution, ordered by best primal
333 // objective first.
334 // 2. The solutions with a dual feasible solution, ordered by best dual
335 // objective (unknown dual objective is worst)
336 // 3. All remaining solutions can be returned in any order.
337 repeated SolutionProto solutions = 3;
338
339 // Directions of unbounded primal improvement, or equivalently, dual
340 // infeasibility certificates. Typically provided for TerminationReasonProtos
341 // UNBOUNDED and DUAL_INFEASIBLE
342 repeated PrimalRayProto primal_rays = 4;
343
344 // Directions of unbounded dual improvement, or equivalently, primal
345 // infeasibility certificates. Typically provided for TerminationReasonProto
346 // INFEASIBLE.
347 repeated DualRayProto dual_rays = 5;
348
349 // Statistics on the solve process, e.g. running time, iterations.
350 SolveStatsProto solve_stats = 6;
