24// Solver classes for the weighted set covering problem.
25//
26// The solution procedure is based on the general scheme known as local search.
27// Once a solution exists, it is improved by modifying it slightly, for example
28// by flipping a binary variable, so as to minimize the cost.
29// But first, we have to generate a first solution that is as good as possible.
30//
31// The first solution is then improved by using local search descent, which
32// eliminates the T_j's that have no interest in the solution.
33//
34// A mix of the guided local search (GLS) and Tabu Search (TS) metaheuristic
35// is also provided.
36//
37// The term 'focus' hereafter means a subset of the T_j's designated by their
38// indices. Focus make it possible to run the algorithms on the corresponding
39// subproblems.
40//
41// TODO(user): make the different algorithms concurrent, solving independent
42// subproblems in different threads.
43// TODO(user): implement Large Neighborhood Search
44//
45// An obvious idea is to take all the T_j's (or equivalently to set all the
46// x_j's to 1). It's a bit silly but fast, and we can improve on it later using
47// local search.
48class TrivialSolutionGenerator {
49 public:
50 explicit TrivialSolutionGenerator(SetCoverLedger* ledger) : ledger_(ledger) {}
51
52 // Returns true if a solution was found.
53 // TODO(user): Add time-outs and exit with a partial solution. This seems
54 // unlikely, though.
55 bool NextSolution();
56
57 // Computes the next partial solution considering only the subsets whose
58 // indices are in focus.
59 bool NextSolution(const std::vector<SubsetIndex>& focus);
60
61 private:
62 // The ledger on which the algorithm will run.
63 SetCoverLedger* ledger_;
64};

87// The first solution is obtained using the Chvatal heuristic, that guarantees
88// that the solution is at most 1 + log(n) times the optimal value.
89// Vasek Chvatal, 1979. A greedy heuristic for the set-covering problem.
90// Mathematics of Operations Research, 4(3):233-235, 1979.
91// http://www.jstor.org/stable/3689577
92//
93// Chvatal's heuristic works as follows: Choose the subset that covers as many
94// remaining uncovered elements as possible for the least possible cost per
95// element and iterate.
96//
97// The following paper dives into the details of this class of algorithms.
98// Neal E. Young, Greedy Set-Cover Algorithms (1974-1979, Chvatal,
99// Johnson, Lovasz, Stein), in Kao, ed., Encyclopedia of Algorithms. Draft at:
100// http://www.cs.ucr.edu/~neal/non_arxiv/Young08SetCover.pdf
101//
102// G. Cormode, H. Karloff, A. Wirth (2010) "Set Cover Algorithms for Very Large
103// Datasets", CIKM'10, ACM, 2010.
104class GreedySolutionGenerator {
105 public:
106 explicit GreedySolutionGenerator(SetCoverLedger* ledger)
107 : ledger_(ledger), pq_(ledger_) {}

158// As usual and well-known with local search, SteepestSearch reaches a local
159// minimum. We therefore implement Guided Tabu Search, which is a crossover of
160// Guided Local Search and Tabu Search.
161//
162// Guided Local Search penalizes the parts of the solution that have been often
163// used. It behaves as a long-term memory which "learns" the most used
164// features and introduces some diversification in the search.
165//
166// C. Voudouris (1997) "Guided local search for combinatorial optimisation
167// problems", PhD Thesis, University of Essex, Colchester, UK, July, 1997.
168//
169// Tabu Search makes it possible to degrade the solution temporarily
170// by disallowing to go back for a certain time (changes are put in a "Tabu"
171// list).
172//
173// Tabu behaves like a short-term memory and is the intensification part of the
174// local search metaheuristic.
175//
176// F. Glover (1989) "Tabu Search – Part 1". ORSA Journal on Computing.
177// 1 (2):190–206. doi:10.1287/ijoc.1.3.190.
178// F. Glover (1990) "Tabu Search – Part 2". ORSA Journal on Computing.
179// 2 (1): 4–32. doi:10.1287/ijoc.2.1.4.
180class GuidedTabuSearch {
181 public:
182 explicit GuidedTabuSearch(SetCoverLedger* ledger)
183 : ledger_(ledger),
