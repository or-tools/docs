25// Quadratic assignment problem (QAP) is a classical combinatorial optimization
26// problem. See https://en.wikipedia.org/wiki/Quadratic_assignment_problem.
27// In short, there are a set of n facilities and a set of n locations.
28// For each pair of locations, a `distance` is specified and for each pair of
29// facilities a `weight` is specified (e.g., the amount of supplies transported
30// between the two facilities). The problem is to assign all facilities to
31// different locations with the goal of minimizing the sum of the distances
32// multiplied by the corresponding flows.
33struct QapProblem {
34 // `weights[i][j]` is the amount of flow from facility i to facility j.
35 std::vector<std::vector<int64_t>> weights;
36
37 // `distances[i][j]` is the distance from location i to location j.
38 std::vector<std::vector<int64_t>> distances;
39
40 // Best known solution (-1 if not defined).
41 int64_t best_known_solution = -1;
42
43 // For testing.
44 bool operator==(const QapProblem& q) const {
45 return weights == q.weights && distances == q.distances;
46 }
47};
48
49// Reads a QAP problem from file in a format used in QAPLIB. See
50// anjos.mgi.polymtl.ca/qaplib/ for more context. The format is "n W D", where
51// n is the number of factories/locations, and W and D are weight and
52// distance matrices, respectively. Both W and D are square matrices of size
53// N x N. Each entry of the matrices must parse as double (we CHECK if it
54// does). Multiple spaces, or '\n' are disregarded. For example:
55//
56// 2
57//
58// 0 2
59// 3 0
60//
61// 0 2
62// 1 0
63//
64// defines a problem with two factories and two locations. There are 2 units of
65// supply flowing from factory 0 to factory 1, and 3 units of supply flowing
66// from factory 1 to 0. The distance from location 0 to location 1 is equal
67// to 2, and the distance from location 1 to 0 is equal to 1.
68QapProblem ReadQapProblemOrDie(absl::string_view filepath);
