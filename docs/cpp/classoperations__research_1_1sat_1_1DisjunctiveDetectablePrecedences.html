bool operations_research::sat::DisjunctiveDetectablePrecedences::Propagate ( )
finalvirtual

This will be called after one or more literals that are watched by this propagator changed. It will also always be called on the first propagation cycle after registration.

Split problem into independent part.

The "independent" window can be processed separately because for each of them, a task [start-min, end-min] is in the window [window_start, window_end]. So any task to the left of the window cannot push such task start_min, and any task to the right of the window will have a start_max >= end_min, so wouldn't be in detectable precedence.

Note
the helper returns value assuming the task is present.
