25// Applies presolve steps to improve the MIP -> IP imperfect conversion. The
26// stricter the domain of the variables, the more room we have for scaling the
27// constraint to integers and prevent overflow. Similarly if we can remove
28// singleton continuous variables, it is just good to do so.
29//
30// Returns the presolve status which can currently be:
31// - INIT for most cases were nothing was proven during this step.
32// - PRIMAL_INFEASIBLE if the model was proven infeasible.
33// - INFEASIBLE_OR_UNBOUNDED if the presolve couldn't distinguish between these
34// two statuses.
35// - ABNORMAL if an error occurred.

37 const glop::GlopParameters& glop_params, MPModelProto* model,
38 std::vector<std::unique_ptr<glop::Preprocessor>>* for_postsolve,
39 SolverLogger* logger);
