◆ GenerateCut()

bool operations_research::sat::FlowCoverCutHelper::GenerateCut ( const SingleNodeFlow &  data)

Try to generate a cut for the given single node flow problem. Returns true if a cut was generated. It can be accessed by cut().

Reference: "Lifted flow cover inequalities for mixed 0-1 integer programs". Zonghao Gu, George L. Nemhauser, Martin W.P. Savelsbergh. 1999.

We are looking for two subsets CI (in-flow subset) and CO (out-flow subset) so that sum_CI capa - sum_CO capa = demand + slack, slack > 0.

Moreover we want to maximize sum_CI bool_lp_value + sum_CO bool_lp_value.

Start by selecting all the possible in_flow (except low bool value) and all the out_flow with a bool value close to one.

This is the best slack we can hope for.

Now greedily remove item from the in_cover and add_item to the out_cover as long as we have remaining slack. We prefer item with a high score an low slack variation.

Note
this is just the classic greedy heuristic of a knapsack problem.

Sort by decreasing score.

Greedily remove/add item as long as we have slack.

The non-lifted simple generalized flow cover inequality (SGFCI) cut will be demand - sum_CI flow_i - sum_CI++ (capa_i - slack)(1 - bool_i)

  • sum_CO capa_i + sum_L- slack * bool_i + sum_L– flow_i >=0

Where CI++ are the arc with capa > slack in CI. And L is O \ CO. L- arc with capa > slack and L– the other.
